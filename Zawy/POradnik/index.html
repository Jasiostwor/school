1. Wprowadzenie do Props

Czym są Props?

Props (skrót od "properties") to parametry przekazywane do komponentów Reacta, które pozwalają na dostosowanie ich zachowania i wyglądu. Są one podobne do argumentów funkcji w JavaScript.

Przekazywanie Props

Props są przekazywane do komponentów jako atrybuty w JSX.

// Definicja komponentu
function Greeting(props) {
  return <h1>Witaj, {props.name}!</h1>;
}

// Użycie komponentu z props
<Greeting name="Jan" />;

Props vs State

  • Props: Są przekazywane do komponentu z zewnątrz i są niezmienne w kontekście komponentu.
  • State: Jest zarządzany wewnątrz komponentu i może być zmieniany w czasie.

2. Przykłady Użycia Props

Przykład 1: Prosty Komponent z Props

function Welcome(props) {
  return <h1>Witaj, {props.username}!</h1>;
}

// Użycie komponentu
<Welcome username="Anna" />;

Przykład 2: Domyślne Wartości Props

Możesz zdefiniować domyślne wartości dla props w przypadku, gdy nie zostaną one przekazane.

function Button(props) {
  return <button>{props.label}</button>;
}

Button.defaultProps = {
  label: "Kliknij mnie",
};

// Użycie komponentu bez przekazania label
<Button />;

Przykład 3: Walidacja Props z PropTypes

Możesz użyć biblioteki PropTypes do walidacji typów props.

import PropTypes from 'prop-types';

function UserProfile(props) {
  return <div>{props.name}</div>;
}

UserProfile.propTypes = {
  name: PropTypes.string.isRequired,
};

3. Wprowadzenie do State

Czym jest State?

State to obiekt, który przechowuje dynamiczne dane komponentu i wpływa na jego renderowanie. W funkcjonalnych komponentach używamy hooka useState do zarządzania stanem.

Inicjalizacja State

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return <div>{count}</div>;
}

Aktualizacja State

Stan aktualizujemy za pomocą funkcji ustawiającej (np. setCount).

<button onClick={() => setCount(count + 1)}>Zwiększ</button>

4. Przykłady Użycia State

Przykład 1: Licznik

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Wartość licznika: {count}</p>
      <button onClick={() => setCount(count + 1)}>Zwiększ</button>
      <button onClick={() => setCount(count - 1)}>Zmniejsz</button>
    </div>
  );
}

Przykład 2: Lista z Możliwością Dodawania Elementów

import React, { useState } from 'react';

function TodoList() {
  const [items, setItems] = useState([]);
  const [inputValue, setInputValue] = useState('');

  const addItem = () => {
    if (inputValue.trim() !== '') {
      setItems([...items, inputValue]);
      setInputValue('');
    }
  };

  return (
    <div>
      <h2>Lista Zadań</h2>
      <input
        type="text"
        placeholder="Dodaj nowe zadanie"
        value={inputValue}
        onChange={(e) => setInputValue(e.target.value)}
      />
      <button onClick={addItem}>Dodaj</button>
      <ul>
        {items.map((item, index) => (
          <li key={index}>{item}</li>
        ))}
      </ul>
    </div>
  );
}

export default TodoList;

Przykład 3: Użycie useState z Obiektami

import { useState } from 'react';

function Profile() {
  const [user, setUser] = useState({
    name: '',
    age: null,
  });

  const updateName = (e) => {
    setUser({
      ...user,
      name: e.target.value,
    });
  };

  const updateAge = (e) => {
    setUser({
      ...user,
      age: e.target.value,
    });
  };

  return (
    <div>
      <input type="text" placeholder="Imię" onChange={updateName} />
      <input type="number" placeholder="Wiek" onChange={updateAge} />
      <p>
        Imię: {user.name}, Wiek: {user.age}
      </p>
    </div>
  );
}

5. Podsumowanie

Zrozumienie props i state jest kluczowe dla efektywnego korzystania z Reacta. Props pozwalają na przekazywanie danych do komponentów, czyniąc je bardziej elastycznymi i wielokrotnego użytku. State umożliwia komponentom zarządzanie własnym stanem i reagowanie na interakcje użytkownika.

Pamiętaj:

  • Props są niezmienne w kontekście komponentu, do którego są przekazywane.
  • State można zmieniać, co powoduje ponowne renderowanie komponentu z nowymi danymi.
  • Przy pracy z obiektami w state, używaj operatora rozproszenia (...) do aktualizacji stanu bez mutowania oryginalnego obiektu.
